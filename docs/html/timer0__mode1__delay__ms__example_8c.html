Detailed Description

Example demonstrating Timer0 Mode 1 blocking delay with millisecond precision.

This example shows how to use Timer0 in Mode 0 (16-bit timer mode) with millisecond-based blocking delays. It demonstrates the timer0_mode1_delay_ms() function which allows specifying the delay duration directly in milliseconds rather than calculating tick values manually.

The example configures Timer0 to run in 1T mode (one clock cycle per count) and uses the blocking delay function to wait for 1 millisecond. The LED on P1.0 toggles every 1000 delay iterations, resulting in a blink rate of once per second (1ms * 1000 = 1000ms = 1s).

Key components demonstrated:

  • Timer0 initialization in Mode 1 with 1T timing (fastest mode)
  • Using timer0_mode1_delay_ms() for blocking millisecond delays
  • LED blinking using precise timing delays
  • Simple loop-based timing control

Hardware setup:

  • LED connected to P1.0 (blinks every 1 second )
Note
The timer0_mode1_delay_ms() function blocks program execution until the specified delay has elapsed. This is suitable for simple timing applications where the CPU can afford to wait.

void main ( void  )

Main function for Timer0 Mode 1 blocking delay example.

This function initializes Timer0 and enters an infinite loop where it performs precise 1ms delays and toggles an LED every 1000 iterations.

Operation flow:

  1. Initialize Timer0 in Mode 1 with 1T timing (one timer count per clock cycle)
  2. Enter infinite loop:
    • Perform DELAY_SCALE (1000) iterations of 1ms blocking delays
    • Toggle LED on P1.0 after each 1000ms interval
    • Repeat indefinitely
Note
The timer runs synchronously (blocking), meaning the CPU waits during each delay period and cannot perform other tasks
The LED blink rate is exactly 1Hz (once per second) with 1ms delays
Using 1T mode provides the highest resolution timing but consumes more power than 12T mode
